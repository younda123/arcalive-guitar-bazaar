# 베타 테스트 체크리스트

## 접속 경로

- 방문자 상품 목록: `/` 또는 `/items`
- 상품 등록: `/items/new`
- 당첨자 코드 입력: `/winner`
- 관리자 페이지: `/admin`

관리자 페이지는 상단 메뉴에 노출하지 않습니다. 운영자가 직접 URL을 입력해 접근합니다.

## 베타 테스트 전 설정

- `.env`의 `ADMIN_PASSWORD`를 실제 비밀번호로 변경
- `docker compose up -d --build` 실행
- `http://127.0.0.1:3002` 접속 확인
- `http://127.0.0.1:3002/admin` 관리자 로그인 확인
- `scripts/backup.ps1` 실행 후 백업 zip 생성 확인

## 테스트 시나리오

1. 후원자 역할로 상품을 1개 등록합니다.
2. 관리자 페이지에서 등록 상품이 `승인 대기` 상태인지 확인합니다.
3. 관리자가 상품을 승인합니다.
4. 방문자 상품 목록에서 승인된 상품이 보이는지 확인합니다.
5. 관리자가 테스트 당첨자를 등록합니다.
6. 당첨자 코드로 접속해 상품을 선택합니다.
7. 선택 완료 후 후원자 연락처가 보이는지 확인합니다.
8. 관리자 페이지에서 상품 상태가 `선택 완료`로 바뀌었는지 확인합니다.
9. 관리자가 `전달 완료`로 처리합니다.
10. 백업을 생성합니다.

## 베타 테스트 중 기록할 것

- 상품 등록 중 막힌 부분
- 이미지 업로드 실패 여부
- 모바일에서 글자나 버튼이 잘리는 부분
- 당첨자 코드 입력에서 헷갈리는 부분
- 관리자 화면에서 찾기 어려운 작업
- 실제 운영자가 추가로 필요한 기능

## 공개 전 주의

- 관리자 비밀번호를 기본값으로 두지 않습니다.
- `data/bazaar.db`와 `public/uploads/`는 삭제하지 않습니다.
- Cloudflare Tunnel 연결은 별도로 확인한 뒤 진행합니다.
