# 남은 작업

## 1. 운영 전 필수 점검

- `.env`의 `ADMIN_PASSWORD`를 실제 운영용 비밀번호로 변경
- `data/bazaar.db`와 `public/uploads/` 백업 경로 확인
- 미니 PC 재부팅 후 `docker compose up -d`로 정상 복구되는지 확인
- 관리자 페이지에서 상품 등록, 승인, 선택 취소, 전달 완료 흐름 확인

## 2. 관리자 기능 보강

- 상품 이미지 교체 기능
- 상품 완전 삭제 전 확인 절차
- 당첨자 삭제 전 확인 절차
- 관리자 목록에서 상태별/검색어별 필터
- 선택 결과 CSV 내보내기

## 3. 사용자 화면 보강

- 모바일 화면에서 상품 등록 폼 확인
- 모바일 화면에서 당첨자 선택 흐름 확인
- 이미지가 없는 상품의 기본 썸네일 개선
- 선택 완료 화면의 연락처 표시를 더 명확하게 정리

## 4. 오류 처리 보강

- 상품 등록 실패 안내
- 이미지 저장 실패 안내
- 중복 당첨자 코드 안내
- 이미 선택된 상품을 다시 선택하려 할 때 안내
- DB 잠금 또는 파일 권한 오류 안내

## 5. 백업과 복구

- `scripts/backup.ps1`을 작업 스케줄러에 등록할지 결정
- 백업 파일 보관 기간 결정
- 실제 복구 리허설 진행
- 외장 디스크 또는 NAS로 백업 복사 여부 결정

## 6. Cloudflare Tunnel 연결

이 작업은 도메인과 기존 Tunnel 설정 확인이 필요하므로 후순위로 둡니다.

- 사용할 hostname 결정
- 기존 Cloudflare Tunnel ingress에 `http://127.0.0.1:3002` 추가
- 실제 도메인에서 관리자 로그인, 이미지 업로드, 당첨자 선택 흐름 확인
- 필요하면 Cloudflare Access로 관리자 경로 추가 보호 검토

## 7. 이벤트 운영 전 최종 확인

- 실제 상품 1개 등록
- 관리자 승인
- 테스트 당첨자 코드 생성
- 테스트 당첨자로 상품 선택
- 선택 후 후원자 연락처 노출 확인
- 전달 완료 처리
- 백업 생성
