# arcalive-guitar-bazaar

아카라이브 기타 채널 커뮤니티 바자회 이벤트 운영을 위한 웹 애플리케이션입니다.

이 프로젝트는 커뮤니티 유저들이 후원한 상품을 등록하고, 이벤트 당첨자가 현재 남아있는 상품 중 하나를 선택한 뒤 후원자와 직접 연락해 상품을 전달받는 흐름을 웹으로 관리하기 위한 도구입니다.

회원가입 기반 서비스가 아니라, 커뮤니티 이벤트 운영을 보조하는 단순 등록/조회/선택 시스템입니다.

---

## 1. 프로젝트 목적

기존 커뮤니티 이벤트 운영에서 수작업으로 처리하던 다음 과정을 웹페이지로 대체합니다.

- 후원 상품 등록
- 상품 리스트 정리
- 상품 상세 정보 조회
- 당첨자별 상품 선택
- 선택 완료 상태 관리
- 선택된 상품의 후원자 연락처 확인

최종 목표는 다음 흐름을 자동화하는 것입니다.

```text
상품 등록 → 상품 공개 → 당첨자 선택 → 후원자 연락 → 전달 완료
```

---

## 2. 핵심 운영 원칙

- 회원가입은 사용하지 않습니다.
- 비로그인 사용자는 상품 목록과 상품 상세 정보를 볼 수 있습니다.
- 상품 등록은 단순 입력식으로 처리합니다.
- 후원자는 상품 등록 시 연락처를 입력합니다.
- 후원자 연락처는 공개 상품 리스트나 상세 페이지에 노출하지 않습니다.
- 당첨자는 운영자가 발급한 코드로 접속합니다.
- 당첨자는 현재 남아있는 상품 중 하나를 선택할 수 있습니다.
- 상품 선택 후에만 해당 상품의 후원자 연락처를 확인할 수 있습니다.
- 상품 전달은 후원자와 당첨자가 외부에서 직접 진행합니다.

---

## 3. 사용자 계층

### 3.1 비로그인 사용자

일반 방문자입니다.

가능한 기능:

- 이벤트 개요 조회
- 상품 리스트 조회
- 상품 상세 조회

불가능한 기능:

- 상품 선택
- 후원자 연락처 조회
- 당첨자 정보 조회
- 상품 수정
- 관리자 기능 접근

---

### 3.2 후원자

상품을 등록하는 사용자입니다.

후원자는 별도 회원가입을 하지 않습니다.

상품 등록 시 다음 정보를 입력합니다.

- 상품명
- 상품 설명
- 상품 상태
- 상품 이미지
- 전달 방식
- 후원자 연락처

등록된 상품은 기본적으로 승인 대기 상태가 됩니다.

---

### 3.3 당첨자

이벤트 결과에 따라 상품 선택 권한을 가진 사용자입니다.

당첨자는 운영자가 발급한 코드를 입력해 본인 선택 페이지에 접근합니다.

가능한 기능:

- 본인 순위 확인
- 현재 선택 가능한 상품 조회
- 상품 선택
- 선택한 상품의 후원자 연락처 확인
- 전달 상태 확인

---

### 3.4 관리자

이벤트 운영자입니다.

가능한 기능:

- 등록된 상품 승인
- 상품 숨김/삭제
- 상품 상태 변경
- 당첨자 정보 등록
- 당첨자 코드 발급
- 선택 진행 상태 확인
- 선택 결과 확인

---

## 4. 상품 상태 정의

상품은 다음 상태를 가집니다.

```text
pending   : 등록됨 / 관리자 승인 대기
approved  : 공개됨 / 선택 가능
selected  : 당첨자가 선택함
completed : 전달 완료
hidden    : 관리자에 의해 숨김 처리됨
deleted   : 삭제됨
```

### 공개 상품 리스트 노출 대상

상품 리스트에는 다음 상태의 상품만 노출합니다.

```text
approved
selected
completed
```

상품 리스트에 노출하지 않는 상태:

```text
pending
hidden
deleted
```

---

## 5. 상품 리스트 페이지

### 5.1 목적

상품 리스트 페이지는 방문자가 현재 이벤트에 등록된 상품을 한눈에 보고, 관심 있는 상품의 상세 페이지로 이동할 수 있게 하는 페이지입니다.

### 5.2 페이지 경로

```text
/items
```

또는 메인 페이지에서 상품 리스트를 바로 보여줄 수 있습니다.

```text
/
```

MVP에서는 메인 페이지에 상품 리스트를 함께 노출해도 됩니다.

---

### 5.3 상품 카드 구성요소

상품 리스트의 각 상품 카드는 다음 정보를 표시합니다.

```text
상품 썸네일
상품명
상품 상태
전달 방식
상세보기 버튼
```

후원자 닉네임은 표시하지 않습니다.

---

### 5.4 상품 상태 표시 문구

상품 상태는 사용자에게 다음 문구로 표시합니다.

```text
approved  → 선택 가능
selected  → 선택 완료
completed → 전달 완료
```

---

### 5.5 상품 정렬 기준

기본 정렬은 다음 순서를 따릅니다.

```text
1. 선택 가능 상품
2. 선택 완료 상품
3. 전달 완료 상품
```

같은 상태 안에서는 등록순으로 정렬합니다.

```text
createdAt ASC
```

---

### 5.6 상품 리스트 필터

MVP에서는 필터 기능은 필수가 아닙니다.

구현한다면 다음 필터만 둡니다.

```text
전체
선택 가능
선택 완료
전달 완료
```

---

### 5.7 카드 클릭 동작

상품 카드를 클릭하거나 상세보기 버튼을 누르면 상품 상세 페이지로 이동합니다.

```text
/items/{itemId}
```

---

### 5.8 리스트 페이지에서 노출하지 않는 정보

상품 리스트에서는 다음 정보를 절대 노출하지 않습니다.

```text
후원자 연락처
당첨자 정보
상품 수정 링크
관리자 메모
내부 관리 ID
```

---

## 6. 상품 상세 페이지

### 6.1 페이지 경로

```text
/items/{itemId}
```

### 6.2 노출 정보

상품 상세 페이지에는 다음 정보를 표시합니다.

```text
상품 이미지
상품명
상품 설명
상품 상태
전달 방식
```

### 6.3 비노출 정보

일반 방문자에게는 다음 정보를 노출하지 않습니다.

```text
후원자 연락처
당첨자 정보
상품 수정 링크
관리자 메모
```

후원자 연락처는 해당 상품을 선택한 당첨자에게만 노출합니다.

---

## 7. 상품 등록 페이지

### 7.1 페이지 경로

```text
/items/new
```

### 7.2 입력 필드

상품 등록 폼은 다음 필드를 가집니다.

```text
상품명
상품 설명
상품 상태
상품 이미지
전달 방식
후원자 연락처
```

### 7.3 전달 방식

전달 방식은 다음 중 하나로 선택합니다.

```text
배송
직거래
협의
```

### 7.4 등록 후 처리

상품 등록 시 다음 처리를 수행합니다.

```text
상품 ID 생성
상품 상태를 pending으로 저장
관리자 승인 전까지 공개 리스트에는 미노출
```

MVP에서는 수정 링크 기능은 후순위로 둡니다.

---

## 8. 당첨자 상품 선택 흐름

### 8.1 접근 방식

당첨자는 운영자가 발급한 코드를 입력해 접근합니다.

```text
/winner
```

코드 입력 후 유효한 코드라면 당첨자 전용 페이지로 이동합니다.

```text
/winner/{winnerCode}
```

---

### 8.2 당첨자 페이지 표시 정보

당첨자 페이지에는 다음 정보를 표시합니다.

```text
당첨자명
순위
선택 가능 여부
현재 남아있는 상품 목록
```

---

### 8.3 상품 선택 조건

당첨자는 다음 조건을 만족할 때 상품을 선택할 수 있습니다.

```text
유효한 당첨자 코드일 것
아직 상품을 선택하지 않았을 것
선택 대상 상품 상태가 approved일 것
```

MVP에서는 순위별 시간 제한이나 턴 제어는 구현하지 않습니다.
관리자가 선택 가능 여부를 직접 관리하는 방식으로 처리합니다.

---

### 8.4 상품 선택 후 처리

당첨자가 상품을 선택하면 다음 처리를 수행합니다.

```text
상품 상태를 selected로 변경
상품에 선택한 당첨자 ID를 저장
당첨자에 선택한 상품 ID를 저장
선택 완료 화면으로 이동
후원자 연락처를 당첨자에게 표시
```

```env

```

## 15. 최종 목표

이 프로젝트의 최종 목표는 커뮤니티 바자회 이벤트에서 운영자가 직접 처리하던 반복 작업을 줄이는 것입니다.

핵심은 복잡한 쇼핑몰을 만드는 것이 아닙니다.

```text
등록된 상품을 보여주고,
당첨자가 하나를 선택하고,
선택 결과를 안전하게 기록하고,
필요한 연락처만 선택자에게 공개하는 것
```

이 범위를 벗어나는 기능은 MVP 이후에 추가합니다.
